How I Deployed a Hugo Site on My Github Page

Mar 8, 2022 2 min read

This is the heading of my first post

 1brew install hugo
 2hugo version
 3hugo new site blog
 4cd blog
 5# add a theme
 6# list of themes: https://themes.gohugo.io/
 7git init.
 8git submodule add https://github.com/razonyang/hugo-theme-bootstrap themes/hugo-theme-bootstrap
 9echo theme = \"hugo-theme-bootstrap\" >> config.toml
10hugo new posts/my-first-post.md
11
12cat >> posts/my-first-post.md <<EOF
13---
14title: "First Post"
15date: 2019-03-26T08:47:11+01:00
16draft: false
17---
18
19# This is the heading of my first post
20EOF
21hugo server
22# or For full rebuilds on change:
23hugo server --disableFastRender
24
25http://localhost:1313/
26
27
28
29Create a file in `.github/workflows/gh-pages.yml` with below content.
30
31mkdir -p .github/workflows
32vim .github/workflows/gh-pages.yml
33
34```yaml
35name: github pages
36
37on:
38  push:
39    branches:
40      - main  # Set a branch to deploy
41  pull_request:
42
43jobs:
44  deploy:
45    runs-on: ubuntu-20.04
46    steps:
47      - uses: actions/checkout@v2
48        with:
49          submodules: true  # Fetch Hugo themes (true OR recursive)
50          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod
51
52      - name: Setup Hugo
53        uses: peaceiris/actions-hugo@v2
54        with:
55          hugo-version: 'latest'
56          # extended: true
57
58      - name: Build
59        run: hugo --minify
60
61      - name: Deploy
62        uses: peaceiris/actions-gh-pages@v3
63        if: github.ref == 'refs/heads/main'
64        with:
65          github_token: ${{ secrets.GITHUB_TOKEN }}
66          publish_dir: ./public

Now add everything to github.

 1echo "# blog" >> README.md
 2git init
 3git add .
 4git commit -m "first commit"
 5git branch -M main
 6git remote add origin https://github.com/sirajudheenam/blog.git
 7git push -u origin main
 8
 9# If everything works fine, then create a gh-pages branch on your github.
10
11git branch -b gh-pages
12
13# If you push anything, then it will build and generate hugo static pages and push to `gh-pages` branch.
